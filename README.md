# Excel自然语言/SQL处理工具 - 设计思想原则

## 产品核心原则

### 火山引擎LLM智能处理原则
1. **降低技术门槛**：用自然语言描述需求，火山引擎LLM智能生成SQL查询
2. **标准化JSON输出**：LLM返回包含业务ID、SQL、业务定义、业务描述的标准JSON格式
3. **学习友好设计**：显示LLM生成的SQL语句，帮助用户理解和学习数据处理逻辑
4. **AI确认机制**：用户可确认、修改或重新生成LLM的执行策略

### 极简MVP设计原则
5. **核心功能聚焦**：专注于Excel→SQLite→自然语言→SQL→结果的核心数据流
6. **避免过度设计**：不做复杂的拖拽编排，保持界面简洁和操作直观
7. **快速上手体验**：新用户能在3分钟内完成第一个数据查询和分析
8. **渐进式功能披露**：根据用户熟练度逐步展示高级功能

### 组件库管理原则
9. **自动组件保存**：每次SQL执行成功后自动保存为业务组件，按时间排序
10. **组件分类管理**：支持业务组件和Excel组件的分类切换和管理
11. **一键复用执行**：选择组件后可快速复用SQL和自然语言描述
12. **执行历史追踪**：记录组件执行历史，支持执行统计和性能分析

### P2P分享预留原则
13. **去中心化分享**：为IPFS或Golang P2P技术预留架构空间
14. **数据隐私保护**：分享功能设计要考虑数据安全和用户隐私
15. **团队协作支持**：预留团队数据分享和协同分析的UI/UX设计
16. **权限管理体系**：为分享权限控制和访问管理预留设计空间

### 火山引擎LLM集成原则
17. **AI生成SQL**：基于火山引擎LLM API实现自然语言转SQL功能
18. **标准化响应**：LLM必须返回包含业务ID、SQL、业务定义、业务描述的JSON格式
19. **重试容错机制**：连续3次解析失败后报错，确保系统稳定性
20. **双模式支持**：支持自然语言模式（默认）和SQL高级模式切换

### 技术架构原则
21. **模块化设计**：各功能模块独立开发，支持后续功能的无缝集成
22. **API优先设计**：为P2P和LLM功能预留标准化的API接口
23. **数据安全第一**：所有数据处理和分享都要符合安全和合规要求
24. **性能优化导向**：确保大数据量处理的性能和用户体验

# Excel 自然语言/SQL处理工具 - 火山引擎LLM驱动

## 产品定位

**基于火山引擎LLM的Excel智能处理工具**：用自然语言描述需求，LLM智能生成SQL查询，自动保存为可复用的业务组件。

### 为什么选择我们的Excel工具？

- **🤖 火山引擎LLM驱动**：基于火山引擎大模型，自然语言智能转换为SQL查询
- **📦 组件库管理**：每次执行自动保存为组件，按时间排序，支持一键复用
- **🔄 双模式支持**：自然语言模式（默认）+ SQL高级模式，满足不同用户需求
- **✅ AI确认机制**：LLM生成执行策略后，用户可确认、修改或重新生成
- **🔗 P2P数据分享**：处理结果可通过P2P网络分享给其他用户（预留功能）
- **💡 极简MVP设计**：专注核心功能，避免过度复杂的界面设计

## 🎯 核心价值主张

**用火山引擎LLM智能生成SQL，用组件库管理提升数据分析复用效率**

### 📁 步骤1：上传Excel文件
- **拖拽上传**：支持.xlsx/.csv文件，自动转换为SQLite数据表
- **数据预览**：显示表结构、行数统计、字段类型识别
- **多文件支持**：支持多Excel文件同时导入和关联处理

### 🤖 步骤2：AI智能处理（默认模式）
- **自然语言输入**："查看各团队续保率排名"、"分析月度销售趋势"
- **火山引擎LLM处理**：调用火山引擎API，智能生成标准化JSON响应
- **JSON解析验证**：自动解析业务ID、SQL、业务定义、业务描述四个字段
- **用户确认机制**：显示AI生成的执行策略，用户可确认、修改或重新生成
- **SQL高级模式**：支持切换到SQL模式，直接编写和执行SQL查询

### ⚡ 步骤3：SQL执行与组件保存
- **SQL执行**：执行LLM生成或用户编写的SQL查询
- **结果验证**：验证查询结果的正确性和完整性
- **自动保存组件**：执行成功后自动保存为业务组件，按时间排序
- **执行历史记录**：记录每次执行的详细信息和性能数据

### 📦 步骤4：组件库管理与复用
- **组件分类浏览**：支持业务组件和Excel组件的分类切换
- **一键复用执行**：选择组件后自动填入SQL和自然语言描述
- **执行统计分析**：查看组件使用频率和执行成功率
- **结果导出分享**：支持Excel导出和P2P分享（预留功能）

### 火山引擎LLM驱动的Excel处理体验

**传统Excel操作**：
```
1. 打开Excel文件
2. 手动编写复杂公式
3. 创建数据透视表
4. 配置筛选和排序
5. 重复处理多个文件
6. 手动整理和分析结果
```

**我们的LLM智能操作**：
```
1. 上传Excel文件（自动转换为SQLite）
2. 自然语言描述："分析各团队月度销售排名"
3. 火山引擎LLM智能生成SQL和业务定义
4. 确认执行策略，一键获得分析结果
5. 自动保存为组件，下次一键复用
```

## 🎯 目标用户

### 主要用户群体
1. **业务分析师**：需要快速将Excel数据转换为SQL查询，进行深度数据分析
2. **数据运营人员**：希望用自然语言描述需求，自动生成数据处理流程
3. **中小企业管理者**：不懂SQL但需要数据洞察，通过自然语言获取业务分析
4. **Excel用户转型**：希望从Excel操作升级到SQL化数据处理
5. **团队协作用户**：需要通过P2P网络分享数据分析结果（预留功能）

### 用户痛点
- **Excel操作复杂**：复杂的公式和数据透视表配置门槛高，学习成本大
- **数据处理效率低**：手动操作耗时长，重复性工作多，容易出错
- **SQL门槛高**：想要深度数据分析但不懂SQL语法和数据库操作
- **团队协作困难**：Excel文件难以实时分享，版本管理混乱
- **自然语言需求表达**：知道要什么结果，但不知道如何用技术手段实现

```sql
    filepath TEXT NOT NULL,

    business_tag TEXT,       -- 用户定义的业务标签
    is_pinned BOOLEAN DEFAULT FALSE, -- 是否置顶
    upload_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    file_size INTEGER,
    status TEXT DEFAULT 'active' -- active, archived
);

-- 业务词汇表
CREATE TABLE business_terms (
    id INTEGER PRIMARY KEY,
    term_name TEXT UNIQUE NOT NULL, -- 业务词汇名称
    description TEXT,               -- 词汇描述
    table_name TEXT NOT NULL,       -- 对应的数据表名
    created_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    usage_count INTEGER DEFAULT 0   -- 使用频率
);

-- DAG 流程配置表
CREATE TABLE dag_workflows (
    id INTEGER PRIMARY KEY,
    workflow_name TEXT NOT NULL,
    workflow_config JSON NOT NULL, -- DAG 节点和连接配置
    created_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    last_executed DATETIME
);
```

#### 查询引擎架构
- **自然语言解析器**：将业务描述转换为 SQL 查询
- **SQL 执行引擎**：直接执行用户输入的 SQL 语句
- **结果缓存系统**：缓存查询结果，提升响应速度
- **DAG 执行引擎**：按照流程图顺序执行数据处理任务

我们的目标是成为跨行业数据分析的智能助手，通过自然语言交互彻底改变传统数据处理方式，让每个人都能轻松进行专业级数据分析，无论其技术背景如何。

### 愿景目标
- **技术愿景**：构建最智能的自然语言数据分析引擎
- **用户愿景**：让数据分析成为每个人的超能力
- **行业愿景**：推动数据分析工具的智能化革命
- **社会愿景**：降低数据分析门槛，促进数据驱动决策的普及

我们相信，数据分析不应该成为业务人员的技术门槛。通过将自然语言与数据查询无缝连接，bubu让每个从业者都能快速获得数据洞察，专注于业务决策而非技术操作。

**目标**：成为跨行业数据分析的标准工具，让数据分析民主化。

---

**bubu** - 让数据分析变得简单而智能 🚀

### 核心理念
> "最好的技术是让复杂的事情变得简单，让专业的工具变得人人可用。"

通过 bubu，我们正在重新定义数据分析的未来：
- 🗣️ **对话式分析**：用自然语言描述需求，获得专业分析结果
- 🧠 **智能化处理**：自动识别数据结构，智能建立关联关系
- 🔄 **动态适应**：支持各行业数据场景，持续学习优化
- 🤝 **协作共享**：构建开放的分析模板生态系统
- 🛡️ **安全可控**：本地部署，数据安全，隐私保护
