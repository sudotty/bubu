# P1 阶段：Excel 简化操作工具 - 组件化流程设计

## 产品定位

**Excel自然语言/SQL处理工具**：专注于用自然语言和SQL简化Excel数据处理，将复杂的Excel操作转换为简单的数据流处理。用户通过自然语言描述需求或编写SQL，系统自动生成对应的数据处理流程，实现Excel数据的智能化处理和P2P分享。

### 核心特性
- **自然语言优先**：用自然语言描述数据处理需求，基于火山引擎LLM智能生成SQL
- **SQL化处理**：支持SQL查询Excel数据，降低数据处理门槛
- **组件库管理**：将执行的SQL保存为业务组件，支持Excel组件和业务组件分类
- **P2P数据分享**：处理结果可通过P2P网络分享给其他用户（预留功能）
- **LLM智能生成**：基于火山引擎API实现自然语言转SQL，返回标准化JSON格式

## 核心概念

### 📊 数据流处理（Data Flow Processing）
**将Excel数据处理抽象为：数据输入→数据清洗→业务查询→结果输出的线性流程**

#### 数据流模式
- **A数据上传**：Excel文件导入，自动转换为SQLite表格
- **数据清洗**：BCD三个SQL清洗步骤，处理数据质量问题
- **业务查询**：D1/D2/D3/D4/D5五个业务SQL查询，基于清洗后数据
- **结果集生成**：所有查询结果保存到数据库，形成最终数据集

### 🗣️ 自然语言处理（Natural Language Processing）
**基于火山引擎LLM API，将自然语言描述转换为标准化SQL查询**

#### 处理模式
- **需求描述**：用户用自然语言描述数据处理需求
- **LLM处理**：调用火山引擎API生成标准化JSON响应
- **JSON解析**：解析返回的业务ID、SQL、业务定义、业务描述四个字段
- **执行确认**：用户确认AI生成的执行策略后执行SQL
- **组件保存**：执行成功的SQL自动保存到业务组件库

### 🔗 P2P数据分享（P2P Data Sharing）
**通过IPFS或Golang P2P技术实现数据处理结果的点对点分享**（预留功能）

#### 分享特性
- **结果分享**：数据查询结果可分享给其他用户
- **分享管理**：查看自己的分享历史和他人的分享内容
- **直接下载**：其他用户可直接查看和下载分享的数据结果
- **隐私保护**：支持权限控制和数据加密

### 1.3 数据流处理模式

#### 📊 标准数据流：A数据→清洗→业务查询→结果集

**示例：销售数据分析流程**
```
A数据上传：
- 销售明细.xlsx → SQLite表 sales_detail
- 产品信息.xlsx → SQLite表 product_info  
- 客户信息.xlsx → SQLite表 customer_info

BCD数据清洗（3个SQL步骤）：
B. 去重清洗：DELETE FROM sales_detail WHERE rowid NOT IN (SELECT MIN(rowid) FROM sales_detail GROUP BY order_id)
C. 数据关联：CREATE TABLE merged_sales AS SELECT s.*, p.product_name, c.customer_type FROM sales_detail s LEFT JOIN product_info p ON s.product_id = p.id LEFT JOIN customer_info c ON s.customer_id = c.id
D. 格式标准化：UPDATE merged_sales SET customer_type = TRIM(UPPER(customer_type)) WHERE customer_type IS NOT NULL

D1-D5业务查询（5个业务SQL）：
D1. 月度销售统计：SELECT DATE_FORMAT(order_date, '%Y-%m') as month, SUM(amount) as total_sales FROM merged_sales GROUP BY month
D2. 产品排名分析：SELECT product_name, SUM(amount) as sales, RANK() OVER (ORDER BY SUM(amount) DESC) as ranking FROM merged_sales GROUP BY product_name
D3. 客户价值分析：SELECT customer_type, AVG(amount) as avg_order_value, COUNT(*) as order_count FROM merged_sales GROUP BY customer_type
D4. 销售趋势预测：SELECT month, total_sales, LAG(total_sales) OVER (ORDER BY month) as prev_month FROM monthly_sales
D5. 异常订单检测：SELECT * FROM merged_sales WHERE amount > (SELECT AVG(amount) + 3*STDDEV(amount) FROM merged_sales)

结果集生成：
- 所有查询结果保存到results数据库
- 生成综合分析报告Excel文件
- 支持P2P分享给团队成员
```

### 1.3 Excel工具处理核心功能

#### 1.3.1 Excel多文件智能关联功能
基于SQL中的多表JOIN操作，抽象为Excel工具处理功能：

**功能描述**：用户上传多个Excel文件后，系统自动识别可关联的字段（如VIN码、ID等）

**Excel增强特性**：
- **自动检测相同列名或相似列名**：智能识别可关联字段
- **提供关联预览，显示匹配率**：预览关联结果和数据质量
- **支持一对多、多对一关联**：灵活的关联方式
- **生成关联后的新Excel文件**：输出整合后的数据表

**实现方式**：
```sql
-- 自动生成的关联SQL模板
CREATE TABLE merged_data AS 
SELECT a.*, b.{关联字段} 
FROM {主表} a 
LEFT JOIN {辅助表} b ON a.{关联字段} = b.{关联字段}
```

#### 1.3.2 Excel数据更新和映射功能
基于SQL中的UPDATE操作，抽象为Excel数据处理功能：

**功能描述**：根据映射表自动更新主表数据

**Excel增强特性**：
- **员工信息批量更新**：处理离职交接、部门调动等场景
- **组织架构重组映射**：支持新旧组织架构的自动映射
- **支持条件更新和批量替换**：基于条件的智能更新
- **提供更新前后对比预览**：确保数据更新的准确性

**实现方式**：
```sql
-- 自动生成的更新SQL模板
UPDATE {主表} 
SET {目标字段} = (
    SELECT {映射字段} FROM {映射表} 
    WHERE {映射表}.{关联字段} = {主表}.{关联字段}
)
WHERE EXISTS (
    SELECT 1 FROM {映射表} 
    WHERE {映射表}.{关联字段} = {主表}.{关联字段}
)
```

#### 1.3.3 Excel动态列添加功能
基于SQL中的ALTER TABLE ADD COLUMN，抽象为Excel列扩展功能：

**功能描述**：根据业务需要动态添加计算列

**Excel增强特性**：
- **基于现有列计算新列**：如是否续保标记、业务分类等
- **支持条件判断列**：实现CASE WHEN逻辑的可视化配置
- **提供公式模板和智能建议**：常用计算公式的快速应用
- **实时预览计算结果**：确保计算逻辑的正确性

**实现方式**：
```sql
-- 动态添加计算列
ALTER TABLE {表名} ADD COLUMN {新列名} {数据类型};
UPDATE {表名} SET {新列名} = 
    CASE
```

```python
def parse_llm_response(llm_response, max_retries=3):
    """
    LLM响应解析算法
    1. 验证JSON格式
    2. 检查必需字段（业务ID、SQL、业务定义、业务描述）
    3. 验证SQL语法
    4. 重试机制处理
    """
    required_fields = ['business_id', 'sql', 'business_definition', 'business_description']
    
    try:
        parsed_data = json.loads(llm_response)
        
        # 验证必需字段
        for field in required_fields:
            if field not in parsed_data or not parsed_data[field]:
                raise ValueError(f"Missing required field: {field}")
        
        # 验证SQL语法
        validate_sql_syntax(parsed_data['sql'])
        
        return {
            'success': True,
            'data': parsed_data,
            'error': None
        }
    except Exception as e:
        return {
            'success': False,
            'data': None,
            'error': str(e)
        }
```
