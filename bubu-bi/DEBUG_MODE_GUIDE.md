# 🔧 调试模式使用指南

## 概述

为了帮助开发者更好地理解和调试LLM处理过程，我们在对话界面中集成了调试功能。这个功能可以显示LLM的原始prompt、响应数据、处理时间等详细信息，但对普通用户是隐藏的。

## 启用调试模式

### 方法1：URL参数
在浏览器地址栏中添加 `?debug=true` 参数：
```
http://localhost:34115?debug=true
```

### 方法2：界面切换
1. 在对话界面右上角找到 🔧 图标
2. 点击切换调试模式
3. 启用后图标会变为 "🔧 调试模式"

### 方法3：浏览器控制台
在浏览器开发者工具控制台中执行：
```javascript
localStorage.setItem('bubu-debug-mode', 'true');
// 然后刷新页面
```

## 调试信息内容

当调试模式启用后，每个AI助手的回复消息都会显示一个 "🔍 Debug" 按钮，点击后可以查看：

### 1. 模型信息
- **模型**: 当前使用的LLM模型名称
- **API端点**: LLM服务的API地址

### 2. 性能指标
- **处理时间**: 从发送请求到收到响应的总时间（毫秒）

### 3. 原始数据
- **原始Prompt**: 发送给LLM的完整提示词，包括用户查询和上下文信息
- **LLM原始响应**: LLM返回的完整JSON响应数据

## 调试面板功能

### 展开/收起
- 点击 "🔍 Debug" 按钮展开调试面板
- 点击面板右上角的 "✕" 按钮收起面板

### 数据格式
- JSON数据会自动格式化显示
- 长文本内容支持滚动查看
- 使用等宽字体便于阅读代码和数据

## 使用场景

### 开发调试
- 检查LLM是否收到了正确的prompt
- 验证LLM响应的数据结构
- 分析处理性能和响应时间
- 排查自然语言处理问题

### 系统监控
- 监控API调用状态
- 检查模型配置是否正确
- 分析用户查询的处理效果

### 问题排查
- 当查询结果不符合预期时，查看原始数据
- 检查prompt是否包含足够的上下文信息
- 验证LLM的置信度和业务逻辑

## 安全注意事项

1. **生产环境**: 调试模式主要用于开发和测试，生产环境中应谨慎使用
2. **敏感信息**: 调试信息可能包含敏感的API响应数据，注意保护
3. **性能影响**: 调试模式会增加一些内存使用，大量调试数据可能影响性能

## 关闭调试模式

### 方法1：界面切换
点击右上角的 "🔧 调试模式" 按钮关闭

### 方法2：浏览器控制台
```javascript
localStorage.removeItem('bubu-debug-mode');
// 然后刷新页面
```

### 方法3：清除URL参数
移除地址栏中的 `?debug=true` 参数并刷新页面

## 技术实现

调试功能的技术特点：
- 🎯 **按需显示**: 只有在调试模式下才显示调试按钮和面板
- 📊 **实时数据**: 显示真实的LLM API调用数据
- 🔒 **用户友好**: 普通用户看不到调试功能，不会影响正常使用
- ⚡ **性能优化**: 调试信息只在需要时才渲染
- 💾 **状态持久**: 调试模式设置会保存在localStorage中

---

*这个调试功能让开发者能够深入了解LLM的工作过程，同时保持用户界面的简洁性。*
