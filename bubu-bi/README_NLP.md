# 历史自然语言查询说明（迁移资料，不是当前产品）

> **Migration source only:** 本文只记录旧 Wails 原型的早期交互与测试设想。当前 BuBu 不直接执行模型生成的 SQL；请以[根目录 README](../README.md)、`PRODUCT_MANIFEST.yaml` 和类型化查询计划为准。以下命令、SQL 示例与功能描述不得作为当前产品承诺。

## 功能概述

bubucel 现已支持自然语言查询功能，您可以用中文描述您的查询需求，系统会自动转换为SQL语句并执行。

## 使用步骤

### 1. 上传数据文件
- 支持 Excel (.xlsx, .xls) 和 CSV 文件
- 文件第一行将作为列名
- 上传后数据会自动导入到数据库

### 2. 切换到自然语言模式
- 在查询面板中点击「自然语言」按钮
- 输入框会显示为自然语言输入模式

### 3. 输入查询描述
支持的查询类型包括：

#### 排名查询
- "薪资最高的5个人"
- "年龄最大的员工"
- "技术部薪资排名前3"

#### 统计查询
- "每个部门的平均薪资"
- "技术部有多少人"
- "总共有多少员工"

#### 筛选查询
- "技术部的所有员工"
- "薪资超过15000的员工"
- "年龄在25到30之间的员工"

#### 趋势分析
- "按入职日期统计人数"
- "各部门薪资分布"

### 4. 查看AI解析结果
- 系统会显示生成的SQL语句
- 包含查询说明和置信度
- 可以查看详细的解析过程

### 5. 使用SQL模板
- 点击「模板」按钮查看预设的SQL模板
- 选择合适的模板快速生成查询

## 测试数据

项目包含了一个测试数据文件 `test_data.csv`，包含员工信息：
- 姓名、年龄、部门、薪资、入职日期
- 可以用来测试各种自然语言查询

## 示例查询

```
自然语言输入："技术部薪资最高的3个人"
生成SQL：SELECT * FROM test_data WHERE 部门 = '技术部' ORDER BY 薪资 DESC LIMIT 3

自然语言输入："每个部门的平均薪资"
生成SQL：SELECT 部门, AVG(薪资) as 平均薪资 FROM test_data GROUP BY 部门

自然语言输入："薪资超过15000的员工数量"
生成SQL：SELECT COUNT(*) as 员工数量 FROM test_data WHERE 薪资 > 15000
```

## 注意事项

1. 确保查询中提到的列名在数据表中存在
2. 数值比较请使用具体数字
3. 如果AI解析不准确，可以切换到SQL模式手动编写
4. 查询历史会保存自然语言和SQL两种类型的记录

## 技术实现

- 基于关键词匹配和模板系统
- 支持中文自然语言处理
- 自动识别表名和列名
- 提供置信度评估
