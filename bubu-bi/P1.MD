# bubucel - P1 产品规划文档

## 产品概述

bubucel 是一个基于火山引擎LLM的智能数据分析工具，专注于提供自然语言到SQL的转换能力，以及组件化的数据查询管理。

## 核心功能

### 1. 智能查询引擎

#### 1.1 自然语言查询（默认模式）
- 基于火山引擎LLM API实现自然语言到SQL的转换
- 支持中文自然语言输入，自动生成对应的SQL查询
- LLM返回标准化JSON格式，包含以下固定字段：
  - `businessId`: 业务ID
  - `sql`: 生成的SQL语句
  - `definition`: 业务定义
  - `description`: 业务描述
- 错误重试机制：连续3次格式错误后报错
- AI执行策略确认：用户可选择是否执行生成的SQL

#### 1.2 SQL查询（高级模式）
- 支持直接输入SQL语句进行查询
- 保留传统SQL编写方式，满足高级用户需求
- 两种模式可自由切换，互不冲突

### 2. 组件库管理

#### 2.1 业务组件
- 保存成功执行的查询为可复用组件
- 按执行时间倒序排列（最新在上）
- 包含SQL语句、业务描述等元信息
- 支持一键复用，自动填充查询内容

#### 2.2 Excel组件
- 预留Excel数据处理组件功能
- 支持组件类型切换和筛选
- 统一的组件管理界面

### 3. 数据处理能力

#### 3.1 查询结果展示
- 表格化展示查询结果
- 支持大数据量的分页显示
- 提供数据导出功能

#### 3.2 DDL操作提醒
- 自动识别DDL类型操作（CREATE、ALTER、DROP等）
- 执行前给予用户明确提醒
- 确保数据安全性

## 技术架构

### 前端技术栈
- React + TypeScript
- Tailwind CSS + DaisyUI
- Wails框架集成

### 后端技术栈
- Go语言
- 火山引擎LLM API集成
- SQLite数据库支持

### LLM集成
- 使用火山引擎提供的LLM API
- 标准化Prompt设计，确保返回格式一致性
- 支持自然语言理解和SQL生成

## 产品特色

1. **智能化**：基于真实LLM API，非Mock数据
2. **组件化**：查询结果可保存为可复用组件
3. **双模式**：自然语言和SQL两种输入方式
4. **安全性**：DDL操作提醒，防止误操作
5. **易用性**：极简UI设计，专注核心功能

## 用户体验设计

### 查询流程
1. 用户输入自然语言描述
2. 系统调用LLM API生成SQL
3. 展示AI解析结果，用户确认执行
4. 执行查询并展示结果
5. 用户可选择保存为组件

### 组件复用流程
1. 打开组件库面板
2. 选择业务组件或Excel组件
3. 点击目标组件
4. 自动填充查询内容
5. 执行查询获得结果
